# 实施计划

1. 在修改前读取桥接层与相关规范，刷新/查询可用的代码关系工具；对待修改符号执行 GitNexus upstream impact。若风险为 HIGH/CRITICAL，先向用户报告。
2. 在桥接层为一次连续按压增加私有滤波状态、限频诊断计数和失败恢复计数；不改变公开头文件接口。
3. 在 `touch_read_with_diagnostics()` 中，保留现有 XPT2046 读取、校准和日志语义，接入首帧直通、三帧中值、候选跳点确认与状态清理。
4. 显式返回零点数处理读取失败/连续无效输入，确保 LVGL 不会长期保留按下状态。
5. 复核触摸校准、旋转、点击和拖动路径；运行格式/静态差异检查及项目 `build.sh`（不存在时使用等效 `idf.py build`）。
6. 使用 RFC2217 刷写并监控：确认 1 MHz 初始化、正常按下/释放、接受样本、人工触发或模拟的异常路径日志，以及无 panic/WDT/重启。
7. 提交前运行 GitNexus `detect_changes()`，确保变更仅覆盖预期触摸输入流程；不接触现有用户未提交的 UI 与任务文件。

## 风险文件

- `components/esp_bms_lvgl_bridge/esp_bms_lvgl_bridge.c`：所有触摸输入都经过此处；阈值过严可能损伤快速滑动，阈值过松则难以压制跳点。

## 验证重点

- 触摸 SPI 宏仍为 1 MHz。
- 触摸校准保存/加载和四个旋转方向未被修改。
- 正常轻点首帧立即通过；连续拖动平滑；单帧跳点不驱动 UI 瞬移。
- 连续读取失败不会卡住 LVGL `PRESSED`。
