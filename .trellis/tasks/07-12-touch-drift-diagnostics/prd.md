# 触摸漂移诊断与抗干扰

## 目标

降低 TPM408 2.8 英寸电阻触摸屏偶发坐标漂移、误触和短时失灵对 LVGL 操作的影响，并通过可复现的串口证据区分软件采样问题与触摸硬件、供电或 SPI 链路故障。

## 已确认事实

- 硬件为 ESP32、ST7789（240×320）与 XPT2046 电阻触摸控制器；触摸引脚为 CLK GPIO25、MOSI GPIO32、MISO GPIO39、CS GPIO33、IRQ GPIO36。
- 当前桥接层以轮询方式使用独立 SPI3 总线，XPT2046 组件默认 SPI 时钟为 1 MHz；见 `components/esp_bms_lvgl_bridge/include/esp_bms_lvgl_bridge.h:53-63` 与 `managed_components/atanisoft__esp_lcd_touch_xpt2046/include/esp_lcd_touch_xpt2046.h:18`。
- 受管 XPT2046 驱动单次读取已丢弃首个坐标样本，并最多采集 `CONFIG_ESP_LCD_TOUCH_MAX_POINTS=5` 个有效样本后取均值；见 `managed_components/atanisoft__esp_lcd_touch_xpt2046/esp_lcd_touch_xpt2046.c:199-249`。
- 桥接层在读取后应用校准映射并限频打印坐标和压力，但没有跨读取周期的稳定性判定、跳点拒绝或异常释放恢复；见 `components/esp_bms_lvgl_bridge/esp_bms_lvgl_bridge.c:517-575`。
- 四点触摸校准与旋转兼容、NVS 持久化已完成并曾在真机确认；本任务不能破坏它。
- 既往启动观察确认 XPT2046 能初始化、能连续上报采样和释放事件，未观察到 panic、看门狗或重启；但尚未捕获用户所述的失灵现场日志。

## 需求

- R1. 保持现有触摸坐标校准、四种旋转、点击和拖动交互的兼容性。
- R2. 对同一次按压中的不可信突跳坐标实施保守过滤，避免其直接驱动 LVGL；过滤不得让正常轻点、滑动条或页面滑动明显变慢。
- R2a. 触摸 SPI 继续使用当前 XPT2046 组件宏的 1 MHz 时钟（已处于用户指定的 1–2 MHz 区间）；不提高时钟，也不为“降速”重复修改现有配置。
- R2b. 采用跨读取周期的中值滤波，并对脱离近期轨迹的单帧坐标作拒绝/保持处理，以消除坐标抖动而非重做静态校准。
- R2c. 底层首次报告触点时，必须等到下一帧落在 16px 稳定范围内才向 LVGL 报告 `PRESSED`；未稳定的轻触/漂移不得激活设置行或手势。
- R2d. 设置列表一旦进入 LVGL 垂直滚动，本次手势必须标记为已消费；滚动结束后的行 `CLICKED` 事件不得打开首个或途经的设置项。
- R3. 在触摸读取错误、压力异常或被拒绝的坐标上提供限频、可操作的诊断日志，不泄露凭据。
- R4. 避免“卡在按下状态”：读取失败或连续异常时，LVGL 必须能在有限时间内看到释放状态。
- R5. 不修改 `managed_components/` 内的第三方 XPT2046 驱动；修复限定在项目桥接层和必要的配置/测试资料。
- R6. 构建、刷写并通过 RFC2217 日志验证启动、读取回调、触摸按下/释放路径和无异常重启。

## 验收标准

- AC1. 校准前后及任一屏幕旋转下，正常轻点和连续拖动仍使用正确坐标。
- AC2. 同一按压中的超出合理速度/位移边界的单帧样本不会造成 UI 瞬移，并产生日志计数或限频诊断。
- AC2a. 在设置列表轻触并发生首帧漂移时，LVGL 保持释放状态，不会自动点击其他选项；稳定触摸最多延后一个输入轮询周期开始。
- AC2b. 从任一设置行开始的垂直滚动结束后，当前行不进入详情页；下一次独立轻点仍能正常进入。
- AC3. SPI 读取失败或连续无效样本不会令 LVGL 长期保持 `PRESSED`；恢复后的下一次按压可正常工作。
- AC4. 串口能区分至少三类事实：SPI 读取失败、底层有效但被软件拒绝的异常坐标、正常已接受样本。
- AC5. `build.sh`（或项目等效 ESP-IDF 构建）和相关静态检查通过；真机启动日志无 panic、WDT 或反复重启。

## 不在范围内

- 更换屏幕、重焊排线、添加硬件滤波或修改供电电路。
- 改动显示渲染、UI 页面布局、触摸校准流程或第三方组件源码。
- 承诺软件能修复触摸面板物理损坏、排线接触不良或电源噪声。
