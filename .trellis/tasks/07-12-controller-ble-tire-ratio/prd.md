# 完善控制器蓝牙与轮胎传动参数

## Goal

完善 FarDriver 控制器的蓝牙选择、轮胎规格与传动比设置，使控制器设置页在开关状态、在线状态和参数来源变化时保持清晰一致，并修正控制器参数解析与车速计算错误。

## Background

- 当前控制器设置已具备主开关、绑定、旧轮胎周长和传动比步进项，但控制器候选设备被内嵌在主卡片中，未复用保护板的独立蓝牙选择流程。
- 当前工作区包含控制器首页、设置滚动和动态卡片收缩等未提交改动；本任务必须保留这些改动并只修改必要位置。
- 现有 FarDriver 参数派生错误地从 `D0/D1/D2` 读取轮胎参数，并把控制器传动比固定为 `1.00`；车速换算结果少 10 倍。
- 现有 NVS 使用 `ctl_wheel` 保存轮胎周长，需要兼容已部署配置。
- TFT 中文字体已由项目自定义点阵字体提供；新增文案必须通过缺字审计。

## Requirements

### 控制器设置页

- “控制器页面显示”关闭时，控制器设置卡片只显示“页面显示、控制器类型”两行，卡片高度紧贴第二行，不保留隐藏项占位。
- “控制器页面显示”开启时，卡片显示完整六行：页面显示、控制器连接、蓝牙选择、控制器类型、轮胎规格、传动比。
- 控制器离线时，轮胎规格和传动比仍显示，但必须使用 `LV_STATE_DISABLED` 置灰且不可进入编辑页。
- 控制器在线且提供有效轮胎/传动参数时，显示控制器值和“控制器同步”来源，字段只读。
- 控制器在线但未提供有效参数时，允许编辑本机备用轮胎规格和传动比。
- 轮胎规格按 `轮辋-扁平比-胎宽` 显示，例如 `12-70-90`；旧配置只有周长时显示“旧周长”，不得伪造规格。
- 控制器类型固定为“远驱”，不可编辑。

### 蓝牙选择

- 点击控制器“蓝牙选择”进入独立子页面，完整复用保护板蓝牙选择的状态栏、刷新按钮、候选列表、候选确认弹层、顶部返回和左边缘返回逻辑。
- BMS 与 Controller 只替换候选数据源、绑定信息和 action，不复制两套渲染器。
- 控制器主卡片下不再显示候选设备列表。
- 候选设备名称和 MAC 在 snapshot 更新、确认弹层与返回过程中保持稳定，不因 RSSI 更新或列表刷新丢失。

### 轮胎规格与传动比编辑

- 轮胎规格编辑页使用三个普通有界滚轮：轮辋 `8–24` 英寸、扁平比 `30–100%` 且步进 5、胎宽 `50–200 mm` 且步进 5。
- 传动比编辑页使用一个普通有界滚轮，范围 `0.50–10.00`，步进 `0.01`。
- 未设置轮胎规格时，编辑页草稿预选 `12-70-90`，只有点击绿色确认按钮才提交。
- 传动比默认 `1.00`。
- 滚轮确认后一次提交绝对设置值；返回或取消直接丢弃草稿，不修改 runtime/NVS。
- 启用 `CONFIG_LV_USE_ROLLER`，且公共合同显式检查 roller 可用性。

### 协议与车速

- FarDriver 扩展参数从 `D2/D3/D4` 读取：扁平比、轮辋、胎宽和 `RateRatio`。
- 轮胎周长按标准规格公式计算；`12-70-90` 的周长约为 `1353 mm`。
- `RateRatio / 60` 换算为传动比，因此 `RateRatio=60` 对应 `1.00`。
- 修复车速少 10 倍的问题，控制器参数优先于本机备用参数。
- 无效帧、CRC 错误、参数块不完整、除零和无效参数不得污染最后有效状态或产生有效车速。
- 不向 FarDriver 回写轮胎规格或传动比。

### 公共合同、运行时与持久化

- 扩展 dashboard snapshot，增加轮辋、扁平比、胎宽、参数来源与本机绝对备用值。
- 新 action 追加到现有 enum 末尾，不改变任何已有 action 数值。
- action event 承载绝对轮胎规格/传动比值；旧 numeric delta action 仅保留兼容，不用于新滚轮提交。
- runtime 默认轮胎规格为未设置，传动比为 `1.00`。
- 新增轮辋、扁平比、胎宽 NVS 键并保留 `ctl_wheel` 兼容：
  - 新字段完整且有效时按规格重新计算周长。
  - 只有旧周长时继续作为遗留备用值并显示“旧周长”。
  - 首次确认新规格后写入新字段并替换旧周长语义。
- 控制器有效参数变化时同步本机显示并只请求一次 NVS 保存，禁止逐帧重复写入。
- 控制器值超出 UI 滚轮范围时仍用于车速计算并只读显示，但不得夹断或覆盖本机备用配置。
- 恢复默认后轮胎规格回到未设置、传动比恢复 `1.00`，控制器绑定仍保留。

### 日志与范围

- 删除临时 `[settings-diag]` 日志。
- 保留一次性的控制器参数同步成功/拒绝同步日志，不输出逐帧高频日志。
- 本任务不包含控制器调参、参数回写、故障码诊断或 Web UI 配置。

## Acceptance Criteria

- [x] 协议自检精确覆盖 `D2/D3/D4` 偏移、`12-70-90 ≈ 1353 mm`、`RateRatio=60 → 1.00`、fallback/控制器优先级和确切车速值。
- [x] 协议自检覆盖 CRC、错误长度/帧头/索引、参数块不完整、除零和滚轮边界外控制器参数。
- [ ] 控制器设置关闭态为 2 行、开启态为 6 行，横竖屏均无空白、重叠或滚动边界回归。
- [ ] 离线规格/传动比置灰；在线无控制器参数时可编辑；在线有控制器参数时显示“控制器同步”并只读。
- [ ] 三滚轮规格页与单滚轮传动比页可确认提交，返回/取消不提交，首次 snapshot 更新不吞掉点击。
- [ ] 控制器蓝牙页复用 BMS 渲染器，刷新、候选确认、顶部返回、左边缘返回和双返回路径均正确。
- [ ] 新规格、传动比和旧 `ctl_wheel` 配置在重启后按兼容规则恢复；控制器参数变化不会触发重复 NVS 写入。
- [ ] 恢复默认清除规格、恢复 `1.00`，保留控制器绑定。
- [x] 字库缺字审计、协议自检、`git diff --check`、完整 ESP-IDF build 与 GitNexus compare 检测通过。
- [ ] RFC2217 刷写后启动、扫描、连接、参数同步和重启持久化路径无 panic、watchdog、高频页面重建或重复 NVS 写入。

## Out of Scope

- 向 FarDriver 写入任何参数。
- 控制器故障码、调参界面和 Web UI 控制器设置。
- 更换控制器首页已选定的视觉方案。
