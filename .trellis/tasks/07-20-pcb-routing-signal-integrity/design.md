# PCB走线信号完整性与四层回流优化 - 技术设计

## 1. 权威边界

- 实时 EasyEDA `PCB1` 是图元、规则、连通性和 DRC 的权威来源。
- `.trellis/tasks/07-16-vehicle-head-unit-hardware/` 和 `docs/hardware/` 是机械、电气意图、风险与验收合同。
- 外部 DSN/SES 和 FreeRouting 产物是候选路径，不是可直接信任的最终设计。

## 2. 层叠与平面

- L1/Top：高速、RF、开关电源局部环路、主要扇出。
- L2_GND：只作连续 GND 平面；在 DSN 中禁止作为可路由层，在 EasyEDA 中校验非 GND 线段数为 0。
- L3_PWR_SIGNAL：电源主干和局部低速信号；避免在 USB/RF/LCD 回流下方形成狭缝或异网岛。
- L4/Bottom：低速信号、产测点扇出和 GND 覆铜；跨层信号就近放回流过孔。

L2 使用 GND 覆铜边框生成完整平面，按已存在的 M3、FPC 槽和天线区域禁布自动避让。L1/L4 建立低优先级 GND 覆铜，配合板边、接口、ESD、RF 和换层点的 GND 缝合过孔。

### 2.1 FPC 槽机械覆盖

- 40Pin 槽固定中心 `(32,33.01)mm`，实体槽 `3×23mm、R1.5`；6Pin 槽固定中心 `(8.715,53.56)mm`，实体槽 `3×6mm、R1.5`。这是 2026-07-20 用户对原 `22mm/5mm` 槽长的明确覆盖。
- 两槽的走线/填充/覆铜/内电层禁布外扩 `0.5mm`，对应圆角禁布包络为 `4×24mm、R2` 与 `4×7mm、R2`；L2_GND 重建后必须自动避让新包络。
- 左、下两侧器件禁布带保持 `1mm`，6Pin 上侧保持 `1mm`；40Pin 上侧受 C16 外框约束保留 `0.9mm`，实体槽到 C16 回读外框仍约有 `0.916mm`，不移动 C16。J2/J3 所在右侧继续保留入口例外。
- 任何因槽外扩引起的既有走线碰撞必须以严格 45°局部重布解决；不得缩小实体槽或 0.5mm 全层无铜区来迁就自动布线。

## 3. 网络类与约束

1. `POWER`：大电流输入、Buck 输入/输出、音频电源，线宽根据电流和局部铜皮分级，不使用普通 8mil 路由。
2. `USB_HS`：两组差分对分别路由，尽量不换层，串阻/保护器件两侧不跨网络等长。
3. `GNSS_RF`：两段 50Ω 单端网络，只走 Top，路径下方 L2_GND 连续，匹配支路最短。
4. `LCD_I80`：串阻后 8 位数据 + WR 为一组，实际布线后以最长线为基准收敛到 5mm 以内。
5. `I2S`：BCLK/LRCK/SDOUT 路径短且同区，避免与 Class-D BTL 和 Buck SW 长平行。
6. `LOW_SPEED`：UART/I2C/GPIO/状态网络，按 3W 和 45°路由，允许 FreeRouting 候选路径。

阻抗几何在没有可制造层叠时只是路由初值。最终文档必须同时记录“当前几何”和“投板前根据阻抗单调整”，不用默认 10/6mil 替代阻抗报告。

## 4. 路由策略

### 4.1 嘉立创EDA自动布线初稿

- 在 RP0 快照完整的前提下，使用文档化的 `pcb_Document.autoRouting()` 参数对当前闭合板框重新运行嘉立创EDA自动布线。
- 自动路由是几何初稿，不是放行结果；允许其不完美，但保留前必须通过层、禁布、线宽、过孔、角度和 DRC 审计。
- 自动结果中 L2_GND 的非 GND 线段必须删除并重布；USB、GNSS RF、Buck/eFuse、LCD WR/数据和 Class-D 不以“已连通”代替人工复核。
- 若 API 明确返回后端不可用，则保留不变更证据并进入 FreeRouting 后备流程，不再围绕同一失败参数循环。

### 4.2 关键网络人工优化

在自动布线初稿上，按下列顺序使用 EasyEDA 原生图元 API 检查、重建并锁定：

1. Buck/eFuse/输入电源热回路和大电流路径。
2. GNSS RF 与天线匹配/偏置网络。
3. USB 两组差分对和短、对称的产测支路。
4. LCD I80 数据/WR 和 I2S。
5. Class-D BTL 输出和本地储能回路。

### 4.3 FreeRouting 后备路由

- 从当前 PCB 导出 DSN，在副本中把 L2_GND 移出可路由层，保留现有禁布对象。
- 只在嘉立创EDA自动布线后端不可用或仍有未布通普通网络时，FreeRouting 使用 Top/Bottom/L3 产生候选路径，不重布已锁定关键网络。
- 解析 SES 后先执行静态校验：网络名存在、层只在白名单、线宽符合网络类、角度规范化为 45°、过孔不落入禁布区。
- 只有通过静态校验的路径才通过 `pcb_PrimitiveLine.create` / `pcb_PrimitiveVia.create` 回写 EasyEDA。

### 4.4 已有 7 个网络

- 保留连通性意图，但删除并重建 6 段非 45°线。
- DBG UART 不要求 TX/RX 等长，但要求更短的跨板路径、更少换层和完整回流。
- 修改后单独运行七网连通检查，防止局部修正恢复飞线。

## 5. 安全性与回滚

- 操作前复制 `.eprj2` 快照，验证 SQLite 完整性和 SHA-256。
- 槽长修改前后分别保留 `pre-slot-length-increase-*` / `post-slot-length-increase-*` 快照；保存重开后回读实体槽、12 个禁布区、132/411/89 图元合同和严格 DRC。
- 每个里程碑保存 EasyEDA 文档；关键网络和普通网络分批回写，每批运行 DRC，出现 Short/Keepout 立即删除本批新增图元。
- 不调用破坏性的整板清除，除非已有快照且删除对象受网络白名单约束。

## 6. 验证数据流

EasyEDA 回读 -> 网络/图元统计 -> 角度/长度/过孔/并行间距分析 -> 严格 DRC -> 保存 -> 关闭/重开 -> 同样的回读和 DRC -> 快照完整性校验 -> 文档同步。

## 7. 主要风险

- 层叠未取得制造阻抗单：允许完成拓扑和初始几何，但不宣称阻抗量产通过。
- 自动路由穿越禁布或分割 L2：使用层白名单、回写前静态校验和回写后 DRC 三道门。
- 密集布局导致无法完成 45°路由：先重试层分配和局部扇出，只在 DRC 证据明确时才提出微调器件。
- 覆铜后产生孤岛或窄颈：检查覆铜图元、GND 连通 DRC 和孤岛保留策略。
