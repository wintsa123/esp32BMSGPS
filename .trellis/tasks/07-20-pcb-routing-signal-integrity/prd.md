# PCB走线信号完整性与四层回流优化

## Goal

在不破坏已冻结机械合同、器件布局和天线净空的前提下，完成 EasyEDA `PCB1` 的四层布线、地平面、关键网络约束和信号完整性收敛，使板卡达到可进入投板前实物/阻抗复核的状态。

## Background

- 用户明确要求创建新任务并实施，本任务采用快速模式，已授权按现有设计证据推进。
- EasyEDA 当前工程为 `esp32BmsGps-Hardware-R1`，PCB UUID 为 `3cbb78f5012cc03c`。
- 实时回读基线为 132 个器件、411 个焊盘、89 个网络、38 段有网络铜线、6 个过孔、0 个覆铜边框、0 个实际覆铜，严格 DRC 为 `Connection Error (358)`。
- 已布通的 7 个局部网络为 `3V3_EXP / EXP_GPIO1 / EXP_UART_TX_GPIO43 / EXP_UART_RX_GPIO44 / GPIO1_EXP_RAW / DBG_UART_TX_GPIO43 / DBG_UART_RX_GPIO44`。
- USB、LCD I80、I2S 和 GNSS RF 尚未布线；当前 USB 已有差分对和等长组，但未绑定可制造层叠的最终 90Ω 几何。
- 现有 38 段铜线中 6 段不满足严格 0°/45°/90°/135° 约束。
- 父任务已证明 EasyEDA 原生自动布线后端不可用；FreeRouting 可产生路径，但直接 SES 回导会破坏 L2_GND 层职责或丢失底层/线宽语义。

## Requirements

### R1 - 保持机械和布局合同

- 保持 `112×68mm` 板框、屏幕投影、4×M3 NPTH、2 个 FPC 槽、12 个机械禁布区、J1/J2/J3/J6 位置和朝向、ESP32 与 GNSS 天线净空。
- 2026-07-20 用户明确覆盖原槽长冻结值：40Pin 槽由 `3×22mm` 加长为 `3×23mm`，6Pin 槽由 `3×5mm` 加长为 `3×6mm`；中心、3mm 宽度和 `R1.5` 圆端不变。槽外 `0.5mm` 全层无铜区同步外扩；器件禁布保持左/下 `1mm`，6Pin 上侧 `1mm`，40Pin 上侧因 C16 实测外框保留 `0.9mm`，右侧继续作为 J2/J3 入口例外。
- 不移动已冻结器件；如 DRC 证明必须微调，先记录冲突和回滚点。

### R2 - 四层层职责与回流

- L1/Top 用于关键高速、RF、器件扇出和局部电源回路。
- L2_GND 为连续 GND 参考平面，禁止普通信号走线，不允许被非机械禁布分割。
- L3_PWR_SIGNAL 优先承载电源面/电源干线，仅在有连续回流路径时承载低速信号。
- L4/Bottom 用于普通信号和底层产测扇出，并使用 GND 覆铜和缝合过孔缩短回流。
- 信号换层时就近放置 GND 回流过孔；高速、RF 和开关节点禁止跨平面分割。

### R3 - 线宽、阻抗、间距和等长规则

- USB D+/D- 按 90Ω 差分目标，连接器侧与 MCU 侧分组、组内尽可能等长、少过孔、无长残桩，线内长度偏差不大于 `0.254mm`。
- GNSS RF 按 50Ω 单端目标，L1 连续参考 L2_GND，不换层，最短连接模组、匹配网络和 U.FL。
- 不在未获得可制造层叠时宣称 90Ω/50Ω 已量产验证；必须记录路由几何所对应的层叠假设和投板前调整项。
- LCD I80 串阻后的 `LCD_DB0_FPC..LCD_DB7_FPC + LCD_WR_FPC` 建立等长/总长约束，数据线与 WR 长度差目标不大于 `5mm`，优先走 Top 并连续参考 L2_GND。
- I2S 三线保持短、避开 Class-D 输出、Buck SW 和 RF；不强制盲目等长，但路径差不得以绕行方式扩大。
- 普通快边沿信号的长平行路径采用至少 3W 间距，优先缩短平行长度；相邻信号层方向优先正交。

### R4 - 布线质量

- 所有直线段只允许 0°/45°/90°/135°，坐标数值误差不得产生可见非 45°斜线。
- 优先修正当前 6 段非规范走线，并减少 DBG UART 跨板绕行和不必要换层。
- 大电流电源和 Class-D 输出短、宽、回路紧凑；Buck CIN-SW-L-COUT 热回路、BST 和 FB Kelvin 路径单独人工复核。
- ESD 泄放顺序、USB 串阻位置、GNSS 匹配位和天线净空不得被自动路由破坏。

### R5 - 工具链与回滚

- 修改前生成可校验的 EasyEDA 工程快照，记录 SHA-256 和 SQLite `integrity_check`。
- 用户明确允许使用嘉立创EDA自动布线作为整板初稿，不要求自动结果一次达到最终最优；自动结果必须再经关键网络人工优化、45°校正和 DRC 收敛。
- 嘉立创EDA自动布线优先，FreeRouting 仅在原生/云端后端仍不可用或有未布通网络时作为后备候选路径；L2_GND 不得保留非 GND 自动路由。
- 不盲信 SES 直接回导；必须校验层、网络、线宽、过孔、45°和禁布区，再通过 EasyEDA 原生图元 API 落板。
- 每个重要阶段保存、关闭、重开并回读，防止只存在于画布内存的未持久状态。

## Acceptance Criteria

- [ ] AC1：EasyEDA 保持 132 个器件、411 个焊盘、89 个网络；板框、M3、天线净空和连接器坐标不变，40Pin/6Pin 槽分别回读为 `3×23mm` / `3×6mm`，配套机械禁布区共 12 个。
- [ ] AC2：L2_GND 不含任何非 GND 普通走线，存在连续 GND 平面，必要的机械避让外无人为分割。
- [ ] AC3：USB 差分对、GNSS RF、LCD I80 和 I2S 的线宽/间距/长度规则存在，实际走线与规则匹配；阻抗层叠假设已记录。
- [ ] AC4：全板有网络的直线段角度检查对非 0°/45°/90°/135° 结果为 0。
- [ ] AC5：所有 89 个网络按原理图意图完成连通，严格 DRC 的 `Connection Error` 为 0。
- [ ] AC6：严格 DRC 无未解释的 Clearance、Keepout、Short、Differential Pair、Length 或过孔规则错误；任何例外都有对象、原因和放行证据。
- [ ] AC7：高速/快边沿信号无跨分割参考面，换层处有就近 GND 回流过孔，长平行信号间距满足约束。
- [ ] AC8：修改后工程快照可打开，SQLite 完整性通过，保存重开后图元计数、规则、长度和 DRC 结果不回退。
- [ ] AC9：`docs/hardware/04-constraints.md`、`06-decisions.md`、`07-schematics.md` 中的 PCB 状态更新为最终实测值，不再声称走线/过孔为 0。

## Out Of Scope

- 不变更原理图功能架构、BOM 主要器件或 GPIO 合同。
- 不以未经实物确认的 FPC 弯曲半径、机壳干涉或量产阻抗报告作为已关闭证据。
- 不移除 `PROVISIONAL / DO NOT FAB` 投板标识，直到布线/DRC 与实物机械门同时通过。
