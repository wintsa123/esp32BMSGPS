# 速度仪表行程记录、零百测试与历史轨迹

## Goal

为速度仪表/控制器页增加统一的运动功能入口，实现可在真机使用的行程记录和 0-100 km/h 加速测试。行程停止后显示统计结果；存在可用 TF 卡时同时保存包含经纬度的轨迹，用户以后可通过设备现有 Web 服务查看历史行程和地图路径。桌面 LVGL 模拟器继续作为横竖屏 UI 与交互验证入口。

## Background

- 当前仓库已有直接编译生产 `esp_bms_lvgl_ui.c` 的 SDL2 桌面模拟器，支持 320x240 和 240x320。
- 当前速度数据已通过 dashboard snapshot 提供，控制器页和 GPS/速度仪表已有现成布局、页面导航和手势逻辑。
- 既有 GPS 父任务已实现连续 RMC 接收、定位状态、速度和 UTC 日期时间，但当前 RMC 解析尚未发布纬度/经度。
- 当前固件已有设备本地 HTTP 服务和统一 `/api/*` 分发，可复用来提供历史行程列表和轨迹数据。
- 开发板自带 TF 卡槽。早期板级定义 `src/board.rs::sd_spi` 与当前硬件规范一致：`MOSI=GPIO23`、`MISO=GPIO19`、`SCK=GPIO18`、`CS=GPIO5`；当前纯 ESP-IDF 固件尚未接入 TF/FATFS。
- 当前 GPS UART1 使用 `RX=GPIO27`、`TX=GPIO18`，因此 GPS UART TX 与板载 TF SCK 的 GPIO 所有权冲突必须在启用卡槽前解决。
- 模态/锁定状态必须用全屏可点击守卫层阻止底层控件穿透。

## Requirements

### 入口遮罩

- 在速度仪表/控制器页上滑或长按时，屏幕中间显示遮罩操作面板。
- 面板仅包含两个大触摸按钮：行程记录的播放图标和 0-100 测试的骷髅头图标。
- 遮罩显示时阻止与下方仪表、分页和快捷面板互动的触摸穿透。
- 点击中央面板外的暗色遮罩区域可关闭面板，不改变当前记录状态。

### 行程记录

- 点击播放按钮后进入记录状态，平均速度和最高速度跟随当前仪表实际显示的速度来源：GPS 仪表使用 GPS，控制器仪表使用控制器速度。
- 轨迹位置始终来自 GPS。GPS 暂时无有效定位时不写入伪造坐标，速度统计和已用时间仍继续；恢复定位后继续记录轨迹点。
- 记录开始后继续停留在原速度仪表/控制器页，以小型红色 `REC + 已用时间` 标记显示当前正在记录，不新增独立的“记录中”页面。
- 记录过程可再次打开操作面板，原播放按钮变为红色停止按钮。
- 点击停止后进入独立结果页，显示平均速度、最高速度、达到最高速度的实际时钟时间（例如 `14:32:08`）和总用时。
- 结果页顶部导航栏左侧提供返回操作，返回原速度仪表/控制器页。

### TF 卡持久化

- 行程记录功能不依赖 TF 卡才能开始；TF 卡不存在或挂载失败时仍完成内存统计和结果页展示，但明确标记本次未保存。
- TF 卡存在且可写时，每次点击播放创建一条独立历史行程，持续保存 GPS 日期时间、纬度、经度、速度和相对已用时间。
- 点击停止时完成记录文件并保存本次平均速度、最高速度、最高速度时间、总用时和有效轨迹点数量。
- 记录过程中 TF 卡被拔出、空间不足或写入失败时不得导致 UI、GPS、BMS、控制器或 Web 服务崩溃；本次统计继续运行，并在停止结果中显示保存失败。
- 异常断电后，已成功写入的完整轨迹点应尽量可恢复；不得因为最后一条记录未正常结束而使其他历史记录无法读取。

### Web 历史轨迹

- 复用设备现有本地 Web 服务，不新增云服务、账号系统或外部后端。
- Web UI 提供历史行程列表，至少显示开始时间、总用时、平均速度、最高速度和保存状态。
- 用户选择一条历史行程后，可查看该次记录的经纬度路径和起终点，并在地图视图中按行驶顺序绘制轨迹线。
- Web API 从 TF 卡按需读取历史数据，不把全部轨迹常驻在 ESP32 内存中。
- TF 卡不存在、记录损坏或轨迹为空时，Web UI 显示明确的中文状态，不返回伪造路径或导致 HTTP 服务不可用。

### 0-100 km/h 测试

- 点击骷髅头按钮后进入独立全屏测试页。
- 首先显示 GPS 尚未连接的等待状态，然后在模拟器中自动进入 GPS 已连接状态。
- GPS 连接后进入全屏 3 秒倒计时，每秒播放模拟提示音并配合明确的视觉变化。
- 倒计时结束后，全屏仅显示 GPS 速度和单一计时，不显示 0-50/0-100 结果卡、状态栏或其他仪表信息。
- 加速页使用纯黑背景：当前 GPS 速度位于画面中间，其下使用小写 `km/h` 单位，再向下留出间距显示计时。
- 加速过程的背景动画为光束从屏幕中心向四周发射，并随时间/速度持续流动，营造车辆向前高速行驶的流光感。
- 模拟速度会继续上升以完整演示测试。
- 首次达到 50 km/h 和 100 km/h 时分别播放提示音，同一阈值不重复播放。
- 首次达到 100 km/h 时，页面下方的单一计时立即停止并定格为最终 0-100 成绩；实时 GPS 速度和流光动画继续更新。
- 测试页支持长按屏幕，或上滑后点击停止按钮，退出测试并返回原仪表页。

## Constraints

- 行程记录、TF 持久化和 Web 历史轨迹必须进入 ESP-IDF 真机固件；模拟器只替代硬件输入来验证相同 UI 状态和结果布局。
- 真机最高速度时间和轨迹点时间优先使用已校验的 GPS UTC/本地时间；GPS 时间不可用时不得伪造有效绝对时间。
- 目标仍是 ESP32-WROOM-32E、4 MB Flash、无 PSRAM；轨迹读取和 HTTP 输出必须分块处理，不能把完整长行程载入 RAM。
- 启用板载 TF 前必须解决 `SCK=GPIO18` 与 GPS UART TX 的所有权冲突，不能让 UART 与 SPI 同时驱动该引脚。
- 模拟器必须继续直接编译生产 LVGL UI，不复制一份仅供桌面的页面实现。
- Web UI 延续仓库已有的嵌入式纯 HTML/CSS/原生 JavaScript，不为地图历史页增加前端构建链。
- 默认界面文案使用中文；英文切换延续现有设备设置入口。
- 模拟器支持 320x240 横屏和 240x320 竖屏，主信息和按钮不得重叠、裁切或移出可见区。
- 新增或更新的 UI 预览图必须放在仓库根目录 `preview/`。

## Acceptance Criteria

- [ ] 横屏和竖屏中，速度仪表/控制器页均能通过上滑和长按打开中央遮罩，并可关闭而不误触底层。
- [ ] 行程记录跟随当前仪表速度来源，开始后显示 `REC + 用时`；停止后结果页显示平均速度、最高速度、实际时钟时间和总用时，左上角可返回。
- [ ] 无 TF 卡时行程仍可开始、停止并显示统计结果，结果页明确提示未保存；有可写 TF 卡时保存独立历史文件及有效 GPS 经纬度轨迹点。
- [ ] TF 卡拔出、空间不足、写入失败或异常断电不会令固件崩溃，也不会破坏其他已完成历史行程。
- [ ] Web UI 可列出 TF 卡中的历史行程并打开单次详情，在地图视图中绘制按时间排序的经纬度路径；缺卡、空轨迹和损坏记录均有明确状态。
- [ ] 长轨迹通过流式/分块方式读取和响应，在无 PSRAM 的目标上不分配与整条轨迹大小成比例的内存。
- [ ] 0-100 测试可完整经过 GPS 等待、3 秒有声倒计时、全屏 GPS 速度与流光加速、50/100 km/h 单次提示音、100 km/h 计时定格和退出流程。
- [ ] 测试页的长按退出和上滑停止均可用，不会意外触发仪表分页或快捷面板。
- [ ] `./scripts/run-lvgl-simulator.sh --headless` 和 `./scripts/run-lvgl-simulator.sh --headless --portrait` 均通过，且覆盖两条新交互流程的模拟状态转换。
- [ ] 桌面可视模拟器中完成两种方向的手势、布局、动画、按钮状态和文案检查，预览截图保存到 `preview/`。

## Out Of Scope

- 云端账号、远程同步、车队管理或第三方地图账号体系。
- 在 TF 卡中预装或下载完整离线地图瓦片。
- 保存 0-100 测试历史；本期持久化对象仅为用户主动开始/停止的行程记录。
- GPX/KML 导入导出、轨迹编辑或轨迹分享。

## Open Questions

- GPS UART TX 与板载 TF SCK 的 `GPIO18` 冲突采用 GPS TX 改脚、禁用 GPS 发送能力还是其他明确硬件方案。
- Web 地图在手机直连设备 Setup AP、浏览器通常无法访问互联网时，需要怎样的底图降级行为。
- 历史记录保留策略、最大文件数量和 TF 卡空间不足时是否自动删除最旧记录。
- 轨迹点采样频率与最小位移阈值。
