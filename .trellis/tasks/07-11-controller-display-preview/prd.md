# FarDriver 控制器设置、连接与首页数据页

## Goal

新增可持久化的 FarDriver 控制器连接设置和首页控制器数据页，在不破坏现有 BMS、GPS、手机蓝牙与设置手势的前提下，仅采集稳定当前页面需要的主动遥测。

## Background

- 目标硬件为 ESP32-WROOM-32E，ESP-IDF 5.5.4，LVGL 9.5.0，无 PSRAM。
- 首页当前为电池、GPS 两页，页面映射和滚动收尾分别被 GitNexus 标记为 HIGH 和 CRITICAL。
- 用户最终指定 `preview/controller_display_v2_dual_hero.png` 作为横竖屏视觉基准；
  该基准覆盖此前的 `controller_display_v2_two_number.png` 选择。
- FarDriver 使用 BLE 服务 `FFE0`、特征 `FFEC`，状态帧以 `0xAA` 开头并使用 FarDriver 专用 CRC。
- 当前 NimBLE host 同时承担 BMS central 和手机 peripheral，扫描必须串行化。

## Requirements

- 设置根页新增与“保护板设置”同级的“控制器设置”。
- 控制器设置包含：控制器连接、控制器页面显示、蓝牙选择、固定只读类型“远驱”、轮胎周长、传动比。
- 控制器连接和页面显示默认关闭；轮胎周长和传动比默认未设置。
- “控制器页面显示”是控制器设置首项和主开关；关闭时隐藏并关闭“控制器连接”和“蓝牙选择”，但保留绑定信息；再次开启时自动启用连接并连接已绑定设备。
- 控制器绑定 MAC 独立于 BMS；扫描、选择和持久化不得覆盖 BMS 绑定。
- 旧 NVS 配置按可选键加载；缺少新键时采用默认值，不使整组设置失效。
- 恢复默认关闭两个开关并清空回退参数，但保留已绑定控制器。
- 首页映射为 `[电池, GPS]` 或 `[电池, 控制器, GPS]`；显示开关在退出设置返回首页时生效。
- 控制器断线不移除分页，只清除连接及遥测有效位，所有数值显示 `-`。
- 横屏为左侧大车速、右侧蓝色大挡位、底部功率/RPM/控制器温度/电机温度；竖屏为上方大车速、中部蓝色大挡位、底部四项辅助数据。
- 控制器页必须包含设计图中的外边框、主区域边框、蓝色挡位面板边框和底部四列分隔线；数字、标题和单位分别排版，不再使用多行标签模拟整列。
- 底部四列的单位必须紧跟在对应数值右侧；挡位面板在真机上必须显示为蓝色，不允许因 RGB/BGR 色序错误显示为橙色。
- 主区域必须显示 `SPEED` 和 `GEAR` 标题，车速使用整数显示且 `km/h`/`mph` 位于主数字右侧。
- 横屏辅助区为一行四列；竖屏辅助区为 2×2，第一行功率/RPM，第二行控制器/电机温度。
- 车速与挡位使用当前固件可承受的最大清晰数字字号/缩放，按 320x240 与 240x320 实际分辨率分别居中；辅助值、标题与单位的层级和颜色遵循设计图。
- `CTRL` / `MOTOR` 标题位于各自温度单元格左上角，温度值与 `C` 在下方同行；`km/h` / `mph` 不得与右侧边框重叠或被裁切。
- 首页轮播每次用户滑动最多切换一页；水平拖动达到页宽 1/5 即切页，不得继续依赖默认半屏吸附阈值。长距离滑动不得跨过中间页，程序化跳页不受此限制。
- 新增中文设置字符加入现有点阵字体，TFT 语言标记仍使用 ASCII `ZH`/`EN`。
- 高频标签使用生命周期固定缓冲和 `lv_label_set_text_static()`，只更新变化值。
- 轮胎周长和传动比使用现有设置组件组成的 `- / +` 步进详情页，不新增 spinbox/textarea。
- 新增纯 C FarDriver 解析模块，支持 EKSR 紧凑帧和通过 55 项 `flash_read_addr` 映射的扩展帧，校验长度、帧头、消息索引和 CRC。
- 高频帧更新 RPM、速度、挡位和基于 Iq/Id 合成电流的估算功率；低频帧更新控制器温度和电机温度。
- 速度参数优先级为控制器 `0xD0` 参数、用户回退参数、无效；速度遵循全局 km/h/mph 设置。
- NimBLE 最大连接数配置为 3；BMS、FarDriver 和手机蓝牙允许常驻。
- BMS 只在稳定电池页按 500 ms 主动轮询；FarDriver 只在稳定控制器页处理高频通知，其他页面仅维持连接。
- 设置页和页面滚动中暂停主动遥测；滚动结束后由主循环读取只读稳定页面状态并切换数据源，不在 LVGL 滚动回调中调用 BLE。

## Acceptance Criteria

- [ ] 两个开关默认关闭、可操作、重启持久化，恢复默认后关闭。
- [ ] 控制器绑定 MAC 独立保存，恢复默认不清除绑定。
- [ ] 轮胎周长和传动比可步进设置、可清空、重启持久化。
- [ ] 页面显示关闭时为电池/GPS 两页，开启后为电池/控制器/GPS 三页。
- [ ] 控制器未连接或任一字段无效时，对应值显示 `-`，分页仍存在。
- [ ] 横竖屏无重叠；外框、主区/挡位区边框、蓝色挡位背景和辅助区分隔线均与 `preview/controller_display_v2_dual_hero.png` 一致。
- [ ] 车速和挡位为主视觉最大字号并在各自区域居中；`km/h`/`mph`、`kW`、`RPM`、`C` 以及 `CTRL`/`MOTOR` 的位置、字号层级和颜色与设计图一致。
- [ ] 底部 `kW`、`RPM`、`C` 均位于对应数值右侧；真机挡位面板与预览均为蓝色。
- [ ] `SPEED` / `GEAR` 标题完整；车速整数和单位同行；竖屏辅助区为 2×2。
- [ ] `CTRL` / `MOTOR` 位于单元格左上角，辅助文字无乱码，`km/h` / `mph` 与右边框保持间距。
- [ ] 三页模式下从电池页长滑到 GPS 时只落在控制器页，反向亦然；两页模式仍可正常切页。
- [ ] 横屏 63px 回弹、64px 切一页；竖屏 47px 回弹、48px 切一页，任意长滑仍只切一页。
- [ ] 在线与离线状态复用同一布局，单个无效字段只把对应数值显示为 `-`，不造成布局位移。
- [x] 解析器自检覆盖两种帧布局、CRC、RPM、挡位、功率、温度、内部参数和无效帧。
- [ ] BMS 仅在稳定电池页轮询，FarDriver 高频数据仅在稳定控制器页投影。
- [ ] 滚动、设置、旋转、快捷面板和返回首页行为无回归。
- [x] `sdkconfig.defaults` 和有效 `sdkconfig` 的 NimBLE 最大连接数为 3。
- [x] 项目构建通过，GitNexus `detect_changes` 报告 controller、BLE、runtime、dashboard 等预期核心流程；整体风险为 CRITICAL。
- [x] 通过固定 RFC2217 桥刷写真机并完成串口启动验证。

## Out Of Scope

- 控制器参数写入、调参、故障码诊断和 Web UI 配置。
- 未拿到真实 FarDriver 时伪造 10 分钟三设备并发结论。
