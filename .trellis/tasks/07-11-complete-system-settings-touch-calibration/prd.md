# 完善系统设置页与屏幕校准

## Goal

完善 TFT“设置 -> 系统”页面：修复中文文字被裁切的问题，为需要交互调整的系统功能补齐独立功能页，并提供可从系统设置主动启动的电阻触摸屏校准流程。

## Confirmed Facts

- 系统设置当前由 `SETTINGS_SYSTEM_ROWS` 静态行生成，位于 `components/esp_bms_lvgl_ui/esp_bms_lvgl_ui.c:1949`。
- “亮度”和“音量”当前使用 `ESP_BMS_LVGL_ACTION_NONE`，只能显示“快捷面板调节”，点击不会进入功能页。
- “调节条位置”“旋转屏幕”“语言切换”“恢复默认”当前直接触发动作，没有独立功能页。
- 通用设置行使用固定行高、固定标签高度和单行文本，长中文没有换行或滚动策略，位于 `components/esp_bms_lvgl_ui/esp_bms_lvgl_ui.c:3458`。
- 系统详情页在横屏使用 48px 行高、竖屏使用 56px 行高，并按固定坐标纵向排列，位于 `components/esp_bms_lvgl_ui/esp_bms_lvgl_ui.c:3525`。
- XPT2046 驱动当前按 0..4096 ADC 范围直接缩放到屏幕坐标；桥接层只处理旋转、交换轴和镜像，没有用户校准参数或校准入口，位于 `components/esp_bms_lvgl_bridge/esp_bms_lvgl_bridge.c:135` 和 `managed_components/atanisoft__esp_lcd_touch_xpt2046/esp_lcd_touch_xpt2046.c:226`。
- 现有显示设置通过运行时 NVS 路径持久化；触摸校准尚无持久化字段。
- TFT 默认中文，英文切换入口必须保留在设备设置内；屏幕语言状态只能使用现有字体可显示的字符。
- 新增或更新的 UI 预览图片必须放在仓库根目录 `preview/`。

## Requirements

- R1. 系统设置根页和子页面中的标题、副标题、状态值在横屏与竖屏下不得被裁切或互相遮挡。
- R2. 需要连续调节或多步交互的系统功能应进入独立功能页，并复用现有设置导航和返回手势。
- R3. 仍适合单击执行的系统动作不增加无意义的中间页。
- R4. 系统设置中新增“屏幕校准”入口。
- R5. 校准流程必须由用户主动启动，不得在每次启动时强制出现。
- R6. 校准流程必须显示清晰触点目标、采集多个屏幕位置、拒绝明显无效样本，并允许取消而不覆盖旧校准。
- R7. 校准成功后立即应用，并持久化到 NVS，重启后继续生效。
- R8. 校准映射必须与现有四方向屏幕旋转兼容，不能破坏当前触摸方向逻辑。
- R9. 恢复默认设置时应明确处理触摸校准数据，避免用户无法回到出厂映射。
- R10. 改动后的系统设置页与校准页必须生成并检查 `preview/` 下的预览图。
- R11. 最终必须构建、烧录并通过 RFC2217 真机验证触摸采点、保存和重启后的映射。
- R12. 亮度、音量、调节条位置和屏幕校准进入独立功能页。
- R13. 旋转屏幕和语言切换保留单击立即执行。
- R14. 恢复默认使用确认框，确认后同时清除用户触摸校准。

## Acceptance Criteria

- [ ] AC1. 系统页所有可见中文在 320x240 与 240x320 布局中完整显示。
- [ ] AC2. 需要调节的系统项目可进入对应功能页并完成操作，返回按钮和右滑返回正常。
- [ ] AC3. 单击型动作不会因本次改动增加多余页面。
- [ ] AC4. 点击“屏幕校准”进入全屏校准向导，取消不会改变当前映射。
- [ ] AC5. 完成校准后触摸位置与目标位置一致，四种旋转下坐标方向正确。
- [ ] AC6. 校准数据保存到 NVS，设备重启后仍生效；恢复默认可清除校准并恢复出厂映射。
- [ ] AC7. 预览图位于 `preview/`，并已检查横竖屏文字和控件无重叠。
- [ ] AC8. `git diff --check`、ESP-IDF 构建、GitNexus `detect-changes` 和 RFC2217 真机验证完成。

## Out Of Scope

- 修改 Setup AP、BMS、蓝牙或 Web 设置流程。
- 开机强制校准。
