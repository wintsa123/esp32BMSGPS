# GPS 启动自检与双风格启动动画

## Goal

让同一份开源固件既能运行在带 GPS 的目标硬件上，也能在未安装 GPS 的普通 ESP32 上可靠降级；同时用两套可选择的未来科技风启动动画承载真实启动自检过程。

## Background

- 当前 `GPS_UART_READY` 只表示 UART 参数、引脚和驱动初始化成功，并不能证明 GPS 模块存在（`components/esp_bms_idf_runtime/esp_bms_idf_runtime.c:1970`）。
- 当前速度仪表页固定创建在轮播第二页，投屏页固定在第三页（`components/esp_bms_lvgl_ui/esp_bms_lvgl_ui.c:8934`）。
- 当前已有两套机车仪表 UI：BMW S1000RR 与 Honda Fireblade；控制器监控页不属于机车扫表主题（`components/esp_bms_lvgl_ui/include/esp_bms_lvgl_ui.h:122`）。
- GPS 无定位不等于模块缺失：有效 `RMC,V`、其他校验正确的 NMEA 语句或有效 CASBIN 响应都能证明模块在线。
- TFT 目前只保证 ASCII 字体；启动动画状态文案必须使用 ASCII。

## Requirements

### R1. GPS 启动能力探测

- 启动时进入 GPS `PROBING` 状态，并在 3 秒探测窗口内持续读取 UART。
- 收到任一校验正确的 NMEA 完整语句或有效 CASBIN 帧后，状态变为 `AVAILABLE`；不得要求已经定位。
- UART 初始化失败或探测窗口内没有有效协议响应时，状态变为 `UNAVAILABLE`。
- `UNAVAILABLE` 后仍保留轻量 UART 轮询；后续收到有效协议帧时允许自动恢复为 `AVAILABLE`，无需重启。
- GPS 能力状态必须进入统一 dashboard snapshot，供运行时、LVGL 和未来轨迹/零百功能共用。

### R2. 无 GPS 时安全降级

- GPS 不可用时，设置中的 GPS 速度来源不可选，并明确显示“GPS 未检测到”。运行时也必须拒绝无效的来源切换，不能只依赖 UI 禁用。
- GPS 不可用时，A-GNSS 注入接口返回不可用，不向空 UART 注入数据。
- GPS 可用但未定位时，保留速度仪表页并显示搜索/未定位状态。
- 当且仅当 `GPS 不可用 && 控制器未在线` 时，速度仪表页不参与 LVGL 绘制和轮播布局；投屏页前移，滑动过程中不得出现空白页。
- 当控制器随后上线或 GPS 后续恢复时，速度仪表页自动重新加入轮播；两者再次都不可用时自动退出并安全回到电池页。
- 速度来源的运行时选择按实际能力回退：首选用户配置，首选不可用时选择另一在线来源；两者都不可用时速度保持无效。

### R3. 两套启动动画

- 提供并持久化两种启动动画选择：`电量充能` 与 `机车扫表`；默认使用电量充能。
- 电量充能动画使用黑色背景、青蓝霓虹、分段电池、扫描线/角标和真实启动阶段，具有未来科技感。
- 机车扫表动画必须直接复用现有速度仪表页面与更新路径，不维护第三份仿制仪表：
  - 当前风格为 Honda Fireblade 时使用 Fireblade UI；
  - 当前风格为 S1000RR 时使用 S1000RR UI；
  - 当前风格为控制器监控时回退到 S1000RR 机车 UI。
- 扫表速度按 `0 -> 最大值 -> 0` 运动；动画结束后恢复真实 snapshot，不得把演示数据带入行程、GPS 或控制器状态。
- 启动进度需要映射实际阶段（显示初始化、设置加载、BLE 启动、GPS 探测、系统就绪），而不是单纯固定延时。
- 启动动画状态只使用 ASCII，例如 `POWER ON`、`GPS CHECK`、`GPS READY`、`GPS OFFLINE`、`SYSTEM READY`。

### R4. 设置、兼容与资源约束

- 启动动画入口放在设备“系统”设置内，并写入现有 NVS 命名空间；旧设备缺少新键时采用默认值并保持兼容。
- 保留现有速度仪表风格、速度单位、旋转、控制器绑定及用户当前未提交 UI 修改。
- 不引入矢量引擎、图片资源或新字体；目标 ESP32-WROOM-32E 无 PSRAM 时仍可运行。
- 模拟器应能构造 GPS 可用/不可用和控制器在线/离线组合，并验证两套动画。
- 桌面模拟器与真机的“启动动画”选择页导航栏都应显示播放按钮；点击后完整播放当前选择的动画，结束后自动返回同一选择页。
- 播放预览只消费 UI snapshot，不产生 runtime action、不修改行程数据，也不写入持久化设置；真机与模拟器复用同一套生产 UI 按钮和 timer 路径。

## Acceptance Criteria

- [ ] AC1: 冷启动连接 GPS 但无卫星定位时，3 秒内识别为 `AVAILABLE`，页头不是 `GPS OFF`，速度页仍在轮播中。
- [ ] AC2: 冷启动不接 GPS 时，探测结束为 `UNAVAILABLE`，设置显示 GPS 不可选，A-GNSS 返回 503。
- [ ] AC3: GPS 不可用且控制器离线时，从电池页向右滑直接到投屏页，不创建可见速度绘制，也没有空白中间页。
- [ ] AC4: GPS 不可用但控制器上线时，速度页重新加入并使用控制器速度；控制器离线后再次退出。
- [ ] AC5: GPS 在启动后接入并产生有效 NMEA/CASBIN 帧时，能力自动恢复且速度页重新加入。
- [ ] AC6: 系统设置可选择并持久化“电量充能”和“机车扫表”，重启后使用所选动画。
- [ ] AC7: 电量充能动画呈现分段电池与真实阶段；机车扫表分别能使用现有 S1000RR 和 Fireblade UI 完成 `0 -> max -> 0`。
- [ ] AC8: 两种方向的 LVGL headless 模拟器、自测、ESP-IDF 构建均通过；实机刷写后串口日志证明有 GPS 与无 GPS 分支，TFT 动画正常结束进入主界面。
- [ ] AC9: GitNexus `detect_changes --scope compare --base-ref main` 仅报告预期启动、GPS runtime、LVGL UI、主循环和模拟器流程。
- [ ] AC10: 模拟器与真机均显示播放按钮并可完整预览当前动画；结束后安全回到“启动动画”选择页，模拟器旋转重建也能恢复，且没有设置 action、snapshot 或 NVS 变更。

## Out of Scope

- 本任务不实现尚未落地的地图、轨迹存储或零百计时业务页面；只提供它们后续可直接消费的 GPS 能力状态。
- 不做 GPS 波特率自动扫描或任意品牌私有协议适配；当前硬件仍使用 115200 波特率，并接受标准校验 NMEA 与现有 CASBIN。
- 不把一次无定位或暂时收不到卫星视为模块故障。
