# 配置器菜单左键返回

## Goal

让 `start.ps1` 与 `start.sh` 的交互式配置器支持用左方向键返回上一个菜单，允许用户在不重启配置流程的情况下修正先前选择。例如，已进入输入设备选择后按左方向键，应回到显示屏选择。

## Background

- Windows 的 `start.cmd` 仅转交执行 `start.ps1`；实际按键读取由 PowerShell 完成。
- Linux/macOS 由 `start.sh` 读取终端方向键转义序列。
- 当前两个脚本已支持在仪表选择菜单按左方向键返回功能模块选择，但单选菜单（MCU、显示屏、输入设备及自定义硬件参数）尚未提供同等返回路径。

## Requirements

1. 在交互式终端中，左方向键必须作为“返回上一级”使用，而不是移动选中光标。
2. 返回时必须保留各个已确认选择，重新进入菜单后以该值作为默认值或当前选中项。
3. 标准开发板配置流程必须在整个已进入的单选/多选配置链中支持逐级回退：开发板、显示屏、输入设备、功能模块及仪表。
4. 自定义开发板流程必须在整个配置链中支持逐级回退，包括开发板名称、MCU、Flash/PSRAM 容量、显示屏、显示屏总线、I80 数据宽度、输入设备、输入设备总线、功能模块、仪表及 GPIO 配置。
5. 非交互终端、重定向输入以及 `configure` / `validate` 等命令行调用保持现有行为，不引入方向键依赖。
6. `start.cmd` 保持为 `start.ps1` 的启动包装器，不重复实现菜单状态机。

## Acceptance Criteria

- [ ] 在 Windows PowerShell 交互模式中，输入设备菜单按左方向键回到显示屏菜单，当前显示屏仍被标为默认/当前选择。
- [ ] 在 POSIX 终端交互模式中，输入设备菜单按左方向键回到显示屏菜单，当前显示屏仍被标为默认/当前选择。
- [ ] 继续按左方向键可沿已进入的菜单层级逐级返回，且在模块与仪表菜单之间已有的返回行为仍然成立。
- [ ] 自定义开发板的 MCU、显示屏和输入设备相关子菜单也能够逐级返回，不会遗留不匹配的配置状态。
- [ ] 整个标准与自定义配置链均可连续按左方向键逐级回退；回退后再向前确认时不会使用已失效的派生硬件配置。
- [ ] 现有 `tests/configurator_selftest.sh` 通过，并新增或调整可自动验证的静态/非交互回归断言。

## Out of Scope

- 为非终端 GUI、IDE 集成终端或 `cmd.exe` 自行实现独立按键处理。
- 改变生成配置、构建、烧录或命令行参数的行为。
