# 正式 UI 同步与剩余里程实现

## Goal

将 `simulator/range_preview.c` 中已确认的 BMS 页面和设置页原型迁入正式 LVGL UI，并实现可持久化的预设满电里程与双阶段剩余里程估算，使固件、桌面模拟器和主机测试共享同一套正式逻辑。

## Background

- 当前模拟器通过 `simulator/range_preview.c` 在正式 UI 初始化后注入原型控件；最终模拟器必须直接展示正式 UI，不再依赖该覆盖层。
- 蚂蚁 BMS 已向快照提供 SOC、包电压和剩余容量；行程模块已提供公制/英制平均电耗投影。
- 现有快照 32 位 flag 已用满，剩余里程有效状态必须使用显式字段。
- 默认 TFT 字体只覆盖 ASCII；新增中文文案必须进入 `settings_zh_10/13/16.c` 的生成子集。

## Requirements

### R1. BMS 页面正式 UI

- `BMS INFO` 在顶部水平居中并使用小字号；在线状态 `OK` 放大并在状态区域居中。
- 新增“剩余里程”、数值和 `km` 标签；横屏中 `km` 位于数值下方。
- BMS 在线且 GPS 定位有效时显示整组剩余里程；缺少任一连接时隐藏整组。
- 剩余里程结果有效时显示 `0..9999`；算法输入不足且 SOC 也无效时显示 `--`。
- SOC 显示格式为 `76 %`；电池填充使用 2px 内边距并贴合内部高度。

### R2. 双阶段剩余里程算法

- 预设满电里程范围为 `0000..9999 km`，默认值为 `0100 km`；`0000` 合法。
- 实测平均电耗无效时，使用 `预设满电里程 × SOC ÷ 100` 并四舍五入。
- 实测平均电耗有效时，使用固定公制公式：
  `range_km = pack_voltage_mv × capacity_remaining_mah ÷ (100000 × consumption_deci_wh_per_km)`。
- 实测公式只接受正电耗、有效包电压和有效剩余容量；输入暂时无效时回退到 SOC 预设估算。
- 结果四舍五入并限制在 `0..9999 km`。
- 速度显示单位切换不得改变剩余里程结果。
- 不使用电池化学修正曲线，直接使用 BMS SOC 和容量数据。

### R3. 设置页面

- “保护板设置”新增“预设里程”行，副标题显示当前四位值和 `km`。
- 点击进入固定导航栏的三级页面，提供四个独立 `0..9` roller 和确认按钮；确认发送绝对公里值，返回/取消不保存。
- 速度单位使用独立三级选择页并保持固定导航栏。
- 控制器类型为独立第二张卡；“蓝牙绑定”更名为“控制器连接”。
- 一级设置横屏行高为 52px，标题和副标题作为整体垂直居中。

### R4. 数据契约与持久化

- 新增 `ESP_BMS_LVGL_ACTION_SET_PRESET_RANGE`。
- `esp_bms_dashboard_snapshot_t` 新增 `preset_range_km`、`remaining_range_km` 和 `remaining_range_valid`，并同步结构尺寸断言。
- 设置动作复用 `numeric_delta` 传递绝对公里值，并复用现有有效标志，不新增事件 flag。
- NVS 增加预设里程键；缺键时使用 `0100` 并保存，恢复默认时恢复 `0100`。
- 计算助手放入 `esp_bms_speed_dashboard` 纯 C 模块，供 runtime、测试和模拟器复用。

### R5. 模拟器与字体

- 正式 UI 完成后从模拟器构建和运行路径移除 `range_preview.c` 与专用字体覆盖。
- 模拟器快照和动作处理使用正式新增字段与动作。
- `settings_zh_10/13/16.c` 必须包含正式 UI 中全部中文字符，包括“预设里程”“剩余里程”。
- 新增或更新的预览图保存到仓库根目录 `preview/`。

## Constraints

- 不改变速度来源选择、GPS 迟滞、行程积分、BLE/BMS/控制器连接语义。
- 保持现有 LVGL 锁、事件、静态文本缓冲和旋转布局模式。
- 不新增第三方依赖或新的抽象层。
- 工作区已有未提交修改必须保留；只移除本任务明确替代的模拟器原型覆盖。

## Acceptance Criteria

- [ ] 纯 C 自测覆盖默认 `0100`、合法 `0000`、SOC 四舍五入、实测公式、零/负电耗、上限钳制、实测无效回退、km/h 与 mph 一致性。
- [ ] 横竖屏正式 BMS UI 显示正确，且 BMS/GPS 任一无效时隐藏剩余里程整组。
- [ ] 平均电耗从无效变为有效时，显示立即从 SOC 估算切换到实测值。
- [ ] 四位 roller 的保存、取消、返回和固定导航栏行为正确。
- [ ] 模拟器不再编译或调用专用 range preview 覆盖。
- [ ] 字体检查对三套 settings 字体报告零缺失中文字符。
- [ ] 双方向 headless 模拟器、主机自测、`git diff --check` 和 ESP-IDF 构建通过。
- [ ] GitNexus `detect-changes --scope compare --base-ref main` 仅报告预期符号和流程。
- [ ] 通过固定 RFC2217 端点完成一次刷写，并监控启动/NVS 加载且无 panic/WDT。

## Out Of Scope

- 电池化学曲线、温度/老化修正、历史学习模型或新的电耗采样算法。
- Web UI 的预设里程编辑入口。
- 改变现有 BMS/GPS/控制器硬件协议和连接策略。
