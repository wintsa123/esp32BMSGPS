# 本田火刃仪表主题设计

## Boundaries

- 使用 GPT Image 2 生成 320x240 横屏视觉稿，保存为 `preview/honda-fireblade-dashboard-concept.png`。
- 运行时不读取该 PNG；火刃仪表由共享的 LVGL C 代码动态绘制，模拟器与真机使用同一实现。
- 不引入图片解码器、新依赖、新页面框架或独立主题抽象。

## Style Contract

- 新增仪表样式枚举：`0=宝马 S1000RR`、`1=控制器监控`、`2=本田火刃`。
- 保留 NVS 键 `speed_style` 和旧值 `0/1` 语义，只扩展校验范围到 `2`。
- 设备端完全沿用已保存值；模拟器的 `init_snapshot()` 显式设为火刃。
- 保留 `ESP_BMS_DASHBOARD_FLAG_CONTROLLER_PAGE_ENABLED` 作为“当前是控制器监控”的派生兼容标志，样式真值放在快照的 `speed_dashboard_style` 字段。

## Action Flow

1. 设置行点击后通过新动作 `ESP_BMS_LVGL_ACTION_SET_SPEED_DASHBOARD_STYLE` 发送选中索引，复用 `numeric_delta` 载荷和既有有效位。
2. 模拟器或 IDF runtime 校验 `0..2`，更新快照样式并派生控制器页标志。
3. 真机提交动作后使用现有显示设置保存路径写回 `speed_style`。
4. UI 更新根据快照样式显示宝马绘制层、控制器页或火刃绘制层。

## Fireblade Rendering

- 在现有 GPS/速度页内懒创建一个火刃容器和单个 `LV_EVENT_DRAW_MAIN` 绘制对象。
- 画面以白色面板、浅灰信息区、黑色文字、红色高速区和绿色档位为主，使用约 `65%` 的中央速度圆弧。刻度环的外轮廓、内色带、主/次刻度、数字与红区共用同一旋转原点和角度偏移，确保 `0` 所在的整个起始截面在 `7 点钟方向`；右侧保留大号数字车速和右上电量百分比。
- 左侧信息列不使用线框或图标，每个字段以灰色标题条加白色数值行呈现，标题条与仪表弧边相接。
- 左侧字段为平均能耗、剩余里程、平均速度和日期；平均速度使用旅程派生数据，无效时显示 `-- km/h`。
- 中心档位圆使用比上一版更深的柔和暗色阴影，不增加额外装饰层。
- 车速、单位、档位、时间、日期、控制器/电机温度、SOC 和平均能耗都来自快照；数据无效时显示 `--` 或中性状态。
- 不绘制电池图标、左侧列表图标、`P`、`ABS` 或 `SIDE STAND`。
- 同时支持 320x240 横屏和 240x320 竖屏，用两组固定布局参数，不增加通用响应式布局层。
- 竖屏布局保留顶部状态、中部速度刻度环和底部数据区的三段结构；数据行可改为两列排列，但仍使用灰色标题行与白色数值行，无图标、无边框。

## Compatibility And Rollback

- 旧 NVS 数据不需要迁移；回滚代码后，旧固件遇到值 `2` 会按原有校验拒绝，因此回滚前应先在设置中切回宝马或控制器监控。
- 若新绘制层影响性能，可仅隐藏“本田火刃”选项并保留数据合约，不影响旧两种样式。
