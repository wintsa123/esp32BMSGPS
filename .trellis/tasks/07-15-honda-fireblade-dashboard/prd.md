# 新增本田火刃仪表主题

## Goal

在现有 LVGL 仪表模拟器中新增一套参考 Honda CBR Fireblade 信息屏布局的“本田火刃”仪表样式，让用户可以在设置中切换，并让模拟器默认显示该样式以便验收；设备端不覆盖已有持久化选择。

## Requirements

- 以用户提供的 Honda CBR Fireblade 仪表截图为视觉参考，保留白/灰/黑主体、中央大速度表、右侧大号数字车速、档位与紧凑状态信息层级。
- 中央速度表参照原图使用约 `65%` 圆弧。以中心档位圆为原点，整体旋转速度刻度环，使圆弧的物理起点、`0` 刻度和起始端面同时位于 `7 点钟方向`；其余刻度、外环和红色高速区必须作为一个整体同步旋转，不得只移动数字或指针。
- 速度单位 `km/h` 放在中心档位圆的正上方。
- 左上黑色状态区显示控制器温度和电机温度，不显示电池图标或电量。
- 删除 `ABS` 和 `SIDE STAND` 警告条。
- 日期放在左下角；右上角删除 `P`，改为仅显示电量百分比文字，不显示电池图标。
- 左侧列表不使用外边框、单元格边框或图标；每项使用一行灰色背景标题和一行白色背景数值，右边缘贴近中央仪表。
- 速度表起点移动后，左侧灰色标题条必须相应向右延伸，与新的速度表弧边直接相接，不留空白缝隙。
- 中心档位圆增加更深的阴影，强化与仪表底层的前后层次。
- 左侧列表中的 `GRIP ANGLE` 改为平均速度 `AVG. SPEED`，数值稿使用 `72 km/h`。
- 使用 GPT Image 2 生成适用于项目的仪表视觉素材，最终项目资产不得只留在 Codex 临时目录。
- GPT Image 2 产物作为视觉稿保存在 `preview/`；实际仪表由 LVGL 动态绘制，不将包含固定数值的全屏图片直接嵌入运行时。
- 在设备设置的仪表/UI 样式切换项中新增中文“本田火刃”选项，与现有中文设置页保持一致。
- 设备端保留已有持久化选择；没有保存值时仍按旧有默认处理，不强制迁移为“本田火刃”。
- 模拟器每次启动时默认选中“本田火刃”，便于直接检查新仪表效果。
- 模拟器启动后应默认展示“本田火刃”仪表，并可从设置切换到其他样式再切回。
- 新增或更新的 UI 预览图必须放在仓库根目录 `preview/` 下。
- 视觉验收需同时提供 320x240 横屏与 240x320 竖屏效果图；竖屏可重排信息区，但必须保留已确认的 V6 视觉语言和 `7 点钟方向` 速度刻度起点。
- 视觉稿生成后必须先交由用户检查；在用户明确确认前，不启动仪表、设置或 runtime 代码实施。

## Acceptance Criteria

- [ ] 模拟器首次启动显示“本田火刃”仪表，主要布局和信息层级与参考图一致。
- [ ] 视觉稿中央速度表为约 `65%` 圆弧；圆弧物理起点与 `0` 刻度同时位于 `7 点钟方向`，红色高速区随整个刻度环同步旋转。
- [ ] 左上显示两个温度，左侧列表只含灰色标题行和白色数值行，无边框、无图标；左下显示日期，右上仅显示电量百分比。
- [ ] 左侧标题条与速度表弧边直接相接，列表包含 `AVG. SPEED / 72 km/h`，中心档位圆的阴影明显深于前一版。
- [ ] 画面不含 `ABS` / `SIDE STAND`、`P` 或电池图标。
- [ ] 设置页可见“本田火刃”选项，文案、行高和选中状态与现有两个选项一致。
- [ ] 从“本田火刃”切换到现有仪表并切回时，页面正常重建，无崩溃、空白或明显重叠。
- [ ] 设备端已有宝马/控制器监控选择保持不变；模拟器启动时默认选中“本田火刃”。
- [ ] 模拟器编译和目标化检查通过，最终验收截图位于 `preview/` 。
- [x] 用户已检查并明确批准 GPT Image 2 的 V6 横屏与竖屏视觉稿。
- [ ] 竖屏高清稿与 240x320 缩放稿位于 `preview/`，无裁切、重叠或不可读文字。

## Notes

- 参考图中的 Honda/CBR 商标与整个网页背景不在实现范围，只参考仪表内容的视觉语言和布局。
- 仓库当前没有全屏仪表位图的资源转换与加载管线，模拟器与真机共用 LVGL 绘制实现。
