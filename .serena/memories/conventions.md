# Conventions

- No codebase conventions exist yet; document reality after the initial firmware scaffold lands.
- Existing `.trellis/spec/backend` and `.trellis/spec/frontend` files are unfilled generic templates. Do not treat them as applicable embedded firmware standards.
- Use Trellis planning for broad feature work. For complex tasks, keep requirements in `prd.md`, technical decisions in `design.md`, and execution checklist in `implement.md` before implementation starts.
- For this ESP32 product, future conventions should be firmware-oriented: hardware pin maps, driver boundaries, protocol parsing, display/UI layering, settings persistence, and validation commands.
